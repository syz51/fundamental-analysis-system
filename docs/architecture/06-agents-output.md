# Output Agents

## Overview

Output agents generate the final deliverables of the analysis system. They synthesize findings from all specialist agents into professional documentation and manage ongoing position monitoring.

The two output agents are:

1. Report Writer Agent - Documentation generation
2. Watchlist Manager Agent - Position monitoring and alerts

---

## 1. Report Writer Agent

### Purpose

Generate professional documentation

### Responsibilities

- Synthesize findings from all agents
- Create investment memos
- Produce executive summaries
- Maintain consistent narrative
- Format for different audiences
- Include supporting data and exhibits

### Report Types

**One-Page Company Summaries**:

- Quick overview of business and financials
- Key metrics and ratios
- Initial screening assessment
- Recommendation to proceed or pass

**Full Investment Memos**:

- Comprehensive analysis synthesis
- Business model deep dive
- Financial analysis with projections
- Valuation with scenarios
- Risk factors and mitigants
- Final recommendation with confidence

**Quarterly Updates**:

- Performance vs. thesis
- Material developments
- Updated metrics
- Revised outlook
- Hold/sell/add recommendations

**Decision Records**:

- Complete audit trail of analysis
- Agent findings and debates
- Human gate decisions
- Rationale documentation
- Outcome tracking setup

**Performance Reviews**:

- Post-mortem analysis
- Prediction accuracy assessment
- Lessons learned
- Pattern validation
- Agent performance tracking

### Memory-Enhanced Reporting

The Report Writer leverages memory to:

**Include Historical Context**:

- Confidence scores based on track record
- Historical precedents and pattern matches
- Similar company outcomes
- Agent credibility weighting

**Highlight Learning**:

- Patterns identified during analysis
- Lessons from similar past analyses
- Calibration factors applied
- Uncertainty quantification

**Document Predictions**:

- Specific forecasts for outcome tracking
- Confidence intervals with historical basis
- Key assumptions to validate
- Trigger points for re-evaluation

---

## 2. Watchlist Manager Agent

### Purpose

Ongoing position monitoring

### Responsibilities

- Track analyzed stocks and positions
- Set price alerts and triggers
- Trigger event-based reviews
- Manage DCA recommendations
- Update thesis as needed
- Generate monitoring reports

### Monitoring Activities

**Price Level Tracking**:

- Entry point alerts
- Stop loss monitoring
- Target price notifications
- Support/resistance levels
- Unusual volatility detection

**News and Event Monitoring**:

- Material announcements
- Earnings releases
- Management changes
- Regulatory developments
- Competitive threats

**Periodic Thesis Validation**:

- Quarterly progress checks
- Annual full re-evaluation
- Assumption verification
- Pattern confirmation
- Risk reassessment

**Performance Attribution**:

- Track vs. initial thesis
- Identify surprise factors
- Measure prediction accuracy
- Update agent credibility
- Document lessons learned

**Entry/Exit Signal Generation**:

- Buy signals at target prices
- Sell signals on thesis break
- Add/trim recommendations
- Position sizing adjustments
- Risk-based rebalancing

### Watchlist Types

**Active Watchlist**:

- Current holdings being monitored
- Full thesis tracking
- Active alerts and triggers
- Regular update reports

**Target Watchlist**:

- Approved candidates awaiting entry
- Price level monitoring
- Catalyst tracking
- Entry timing recommendations

**Archive Watchlist**:

- Exited positions
- Outcome tracking
- Learning extraction
- Historical reference

### Memory Integration

The Watchlist Manager uses memory to:

**Pattern Monitoring**:

- Check if current developments match historical patterns
- Alert when recognized warning signs emerge
- Validate thesis assumptions against outcomes
- Update pattern success rates

**Adaptive Alerts**:

- Learn optimal alert thresholds
- Reduce false positives over time
- Prioritize high-impact notifications
- Personalize to user preferences

**Outcome Tracking**:

- Feed results back to knowledge base
- Update agent performance scores
- Validate patterns and predictions
- Contribute to system learning
